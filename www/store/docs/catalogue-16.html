Batch Update Products from CSV
Here you can batch update your products from CSV. Please read the instructions here carefully before updating. It is recommended you back up your store before each batch update in case you need to revert the store back to how it was before the update.
Browse for CSV
Locate CSV to import. CSV MUST contain a header row and only certain fields are supported for the batch update. Fields supported are:

codeID = Current product code OR store product id as displayed on product management page. Click the icon to show store product ID. Used to identify product to be updated.
pName = Product name.
pTitle = Alternative browser display title.
pMetaKeys = Meta keywords.
pMetaDesc = Meta description.
pTags = Product tags, comma delimited.
pDescription = Product full description.
pShortDescription = Product short description.
pCode = New product code if applicable.
pDownload = Is product download. Yes or no.
pDownloadPath = Local/HTTP/FTP path to file.
pEnable = Is product enabled? Yes or no.
pVideo = You Tube Video Code if applicable.
pVideo2 = Vimeo Video Code if applicable.
pVideo3 = Daily Motion Video Code if applicable.
pMultiBuy - Multi buy value if applicable.
countryRestrictions = Restricted countries for product purchase. Comma delimited country IDs.
freeShipping = Is free shipping enabled? Yes or no.
pPurchase = Is product purchase enabled? Yes or no.
pStock = New stock level.
pWeight = Product weight. eg: 1000
pPrice = Product price, no currency symbol. eg: 9.99
minPurchaseQty = Minimum purchase quantity. If applicable. 0 or higher.
maxPurchaseQty = Maximum purchase quantity. If applicable. 0 or higher.
pOffer = Product offer price, no currency symbol. eg: 9.99
pOfferExpiry = Product offer price expiry date if applicable. MUST be 0000-00-00 format.
rwslug = Rewrite slug (if using search engine friendly urls)
expiry = Product expiry date if applicable. MUST be 0000-00-00 format.
pCube = If you are using Maian Cube, enter Maian Cube product ID. Refer to your cube installation.
pGuardian = If you are using Maian Guardian, enter Maian Guardian product ID. Refer to your guardian installation.
dropshipping = ID of drop shipping supplier if applicable. Refer to ID on drop shipping page in admin.

Leave columns blank if you don`t wish to update this field. For example if "pTitle" was blank, product title will NOT be updated. An example of the header row is found here. Use this CSV as a template if you wish. Note that columns do not need to be in the above order, but the names MUST be as specified above (case sensitive). Any columns not identified will be skipped.

Be careful if you have several products with the same code as ALL products would be updated with that code. Use the product ID for uniqueness.
Lines to Read
Must be greater than the longest line (in characters) to be found in the CSV file (allowing for trailing line-end characters).

It became optional in PHP 5. Omitting this parameter (or setting it to 0 in PHP 5.0.4 and later) the maximum line length is not limited, which is slightly slower.

NO commas or period symbols. Examples: 1000, 2000, 3000 etc

Delimited/Enclosed by
Set the field delimiter and field enclosure character. If left blank delimiter defaults to a comma.

Fields are enclosed if the delimiter is found in the string to prevent premature end of delimiter. Example on comma delimited string:

0,1,"The delimiter is , a comma",2,3

If left blank defaults to quotes (").
